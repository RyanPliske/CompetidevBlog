Git Command Line Basics

by Ryan Pliske on January 16, 2016

Git GUI Clients are great. There are tons of them out there.

When I first started using Git, I used Git's Graphical User Interface (GUI) for the mac. It does a decent job satisfying your basic needs, but it's very limited. It also hides the real inner workings of Git that you should really be exposed to. Additionally, there are many helpful advanced techniques you can do using the command line. Overall the command line gives you more control of your local and remote repositories. This blog will just go over the basics to get you started. Let the training begin Luke.

In order to begin, you need to install git to your command line. On Mac this can either be done by simply downloading Xcode or by going here. You'll know you're ready to rock and roll when you type in "git --version" in terminal and terminal responds with your version of git you have installed.
