Git Command Line Basics

by Ryan Pliske on January 16, 2016

Git GUI Clients are great. There are tons of them out there.

When I first started using Git, I used GitHub's GUI. A GUI is software that someone has engineered in order to provide a user with a familiar user interface full of graphical icons and visual indicators as opposed to a text-based command line interface (CLI). Windows is a GUI on top of the MS-DOS command line. In order to spare users from a steep learning curve full of memorizing commands and hard to debug errors, GUIs were developed.

The GitHub GUI in particular worked for me as a beginner. It did a decent job satisfying my basic needs, but it's very limited, especially in a team-based setting. It also hides the real inner workings of Git that you need to be exposed to if you're going to develop software professionally. There are many advanced Git techniques capable via the command line, but this blog will just go over the basics. Let the training begin Luke.

In order to begin, you need to install git to your command line. On Mac this can either be done by simply downloading Xcode or by going here. You'll know you're ready to rock and roll when you type in "git --version" and terminal responds with your version of git you have installed. For me, terminal responds with "git version 2.5.4 (Apple Git-61).""

Let's Rock.

How To Init with Existing Project
cd into the directory
git init
git add .
git commit -m "Initial Commit or whatever message"

Boom done. Now you have a git repo on your local computer. The next step is to setup a remote server. I personally use GitHub, but there are others out there like GitLab, BitBucket, etc. I do this so that I can pull my repository from any computer in the world including my iPhone. If my laptop goes down in flames, there's no way I'm letting my code go down with it.

Whichever you choose there's some way you can create a new repository. I'm not going to list instructions, but once you do setup your repo, you'll find a URL that links to it. You'll need to copy it. Go back to terminal and try:

How To Push Existing Repo To Remote Server
cd into the directory git remote add origin <copy and paste repo URL here>
   -Mine looks like this: git@github.com:me/name.git
   -Note: This is through SSH keys. the same can be accomplished via HTTPS.
git push origin master
   -origin means to server and master is the branch you want to push to

Alternatively, you can clone an existing repository from a server. Your teammates will have to do this if you're the person who initially setup the repo.

How To Clone Existing Repo From Remote Server
cd into the directory git clone <copy and paste repo URL here> <. (optional)>

A common problem with this approach is it forces you to create a new directory. Sometimes I want it to be in the same directory that I'm in. You can do this by simply adding the optional period at the end of the command. Additionally you'll want to configure some things, mainly your username and learn how to do some other things. I'm just going to casually post them below.

Show Git Username
git config user.name
OR
git config --list
Change Git Username
git config --global user.name
How to Make a New Branch
git checkout -b <branchName>
How to Reset a branch equal to another branch
git reset --hard <branchName>
How to Amend a Commit
git commit --amend
How to Amend the Author of a Commit
git commit --amend --author="Author Name"
If needed, push that commit to the server
git push --force
Delete Remote Git Tag
For Example: a tag named "12345"
git tag -d 12345
git push origin :refs/tags/12345
Show what branches exist on your local machine
git branch
Show what branches exist on the server
git branch -a
Grab all the branches and bring them to your local machine
git fetch
Grab all recent changes from server and commit your local changes
git pull

Now that we've covered the basics, I do personally use SourceTree by Atlassian. It is the best Git GUI Client that I've come across. I strongly recommend you pick it up. Additionally I do have another more specialized blog about Git Ignores for when dealing with iOS applications. Git Ignores allow you to ignore files that will exist in your local repository, but won't be pushed to the server. You'll definitely want to check that out here.
