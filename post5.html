Git Ignores

by Ryan Pliske on December 31, 2015

It doesn't matter if you're developing a game using Microsoft Visual Studio, or building an iOS application in Xcode, sooner or later you're going to be developing on a team. Regardless of whether you choose SVN, Git or something else for your version control, there are going to be some files you want to keep on your local machine and never pushed to your shared server.

If you choose to use git for your version control, you can accomplish this by setting up a .gitignore file. This file should be in the same directory that your .git folder is located.

In iOS applications, I find it works best if you ignore your.xcuserstate data. This data contains information like breakpoints that you set or any schemes that you've created.

Additionally, if you're using Cocoapods, I find it best if you don't push your pods directory to your repository. The only file relevant to your pods that you want in your repository is your Podfile and your Podfile.lock. If you end up keeping your /Pods folder in your repository, whenever a teammate installs or updates Cocoapods on their local machine, they'll end up messing up everyones.

I just recently had a friend mess up their build and it took 3 of us to fix the problem. He had updated pods on his local machine, pushed it to the GitLab server and it absolutely demolished their automated project build (we have an automated system at work where our server builds our project and can archive it's .ipa file for our internal app store). Personally I will never use Cocoapods ever again, but that's just my personal opinion.

Lastly, If you've already published your git repository, you will most likely have to edit your git cache. Assuming you're using a unix system, you can do this by:

git rm -r --cached path_to_file/

If you're unfamiliar with a unix command line, you can easily find the path to your folder by typing the command: pwd. Actually you can do some more cool stuff with a unix command line. I'll write another blog about it soon.

Below I've included an example for a .gitignore file:

*.pbxuser
*.mode1v3
*.mode2v3
*.perspectivev3
*.xcuserstate
ProjectName.xcworkspace/
xcuserdata/
# Pods - for those of you using CocoaPods
Pods/
