Git Command Line Basics

by Ryan Pliske on January 16, 2016

Git GUI Clients are great. There are tons of them out there.

When I first started using Git, I used GitHub's GUI. A GUI is software that someone has engineered in order to provide a user with a familiar user interface full of graphical icons and visual indicators as opposed to a text-based command line interface (CLI). Windows is a GUI on top of the MS-DOS command line. In order to spare users from a steep learning curve full of memorizing commands and hard to debug errors, GUIs were developed.

The GitHub GUI in particular worked for me as a beginner. It did a decent job satisfying my basic needs, but it's very limited, especially in a team-based setting. It also hides the real inner workings of Git that you need to be exposed to if you're going to develop software professionally. There are many advanced Git techniques capable via the command line, but this blog will just go over the basics. Let the training begin Luke.

In order to begin, you need to install git to your command line. On Mac this can either be done by simply downloading Xcode or by going here. You'll know you're ready to rock and roll when you type in "git --version" and terminal responds with your version of git you have installed. For me, terminal responds with "git version 2.5.4 (Apple Git-61).""

Let's Rock.

How To Init with Existing Project
cd into the directory
git init
git add .
git commit -m "Initial Commit or whatever message"

Boom done. Now you have a git repo on your local computer. The next step is to setup a remote server. I personally use GitHub, but there are others out there like GitLab, BitBucket, etc. I do this so that I can pull my repository from any computer in the world including my iPhone. If my laptop goes down in flames, there's no way I'm letting my code go down with it.

Whichever you choose there's some way you can create a new repository. I'm not going to list instructions because 1. It's easily Googleable and 2. I don't now which server you'll end up using. But once you do setup your repo, you'll find a URL that links to it. You'll need to copy it. Go back to terminal and try:

How To Push Existing Repo To Remote Server
git remote add origin <copy and paste repo URL here>
git push origin master
(origin means to server and master is the branch you want to push to)
